# RecSys Evaluation Results

**Dataset:** MovieLens 25M subset — 1,000,000 ratings, 10,008 movies, 5,159 users.
**Split:** 90% temporal train / 10% temporal val (no leakage).
**Evaluation:** Full-catalog ranking against all 10,008 movies. Train history masked per user.
**Positives evaluated:** 100,000 val ratings.

## Comparison Table

| Model                       | HR@10  | HR@50  | HR@100 | NDCG@10 | NDCG@100 | MRR    |
|-----------------------------|-------:|-------:|-------:|--------:|---------:|-------:|
| Random (theoretical)        | 0.0010 | 0.0050 | 0.0100 |  0.0004 |   0.0015 | 0.0020 |
| Popularity                  | 0.0257 | 0.1003 | 0.1633 |  0.0107 |   0.0365 | 0.0118 |
| **ALS (128 factors)**       | **0.0337** | **0.1067** | **0.1758** | **0.0163** | **0.0429** | **0.0167** |
| Two-Tower v2 (no genome)    | 0.0276 | 0.1036 | 0.1691 |  0.0133 |   0.0400 | 0.0147 |
| Two-Tower v3 (in-batch only)| 0.0003 | 0.0056 | 0.0170 |  0.0001 |   0.0030 | 0.0012 |
| Two-Tower v3 + random negs  | 0.0204 | 0.0762 | 0.1262 |  0.0098 |   0.0297 | 0.0111 |

## Key findings

1. **ALS wins.** Classical Matrix Factorization with 128 factors beats all Two-Tower variants on every metric. This is consistent with known results in CF literature — simple methods often outperform neural approaches on sparse user-item matrices.

2. **Two-Tower v2 is competitive with Popularity** but does not beat ALS. The model has roughly 3M parameters on just 5,159 users, which means serious overfitting risk even with explicit random negatives.

3. **In-batch-only InfoNCE fails catastrophically** on full-catalog eval (HR@10 = 0.0003, worse than random). The model learns to rank positives among popular items in the batch, but cannot generalize to the long-tail. Explicit random negatives sampled from the full catalog are essential.

4. **Tag Genome features (1128 tags) did not provide lift** sufficient to overcome the user embedding bottleneck at this data scale. The tags encode rich item semantics, but without enough user signal (more users needed), the model cannot map user preferences to genome patterns.

5. **Temperature sensitivity:** τ = 0.07 caused rapid memorization and overfitting. τ = 0.15 gave stable training but still could not beat ALS.

## Production model choice

**Final: Two-Tower v2** (checkpoint `lightning_logs/version_14/checkpoints/best-epoch=1-val_hr_at_10=0.8971.ckpt`).

Rationale: the v3 with explicit negatives is close enough in quality (HR@10 = 0.0204 vs v2's 0.0276) and uses the current model architecture. For production deployment we accept being slightly worse than ALS to demonstrate the Two-Tower architecture in a working system.

## Scale-up run (v5) — MovieLens 25M full

**Dataset:** 5,000,000 ratings, 62,423 movies, 32,564 users.
**Eligibility:** users with 10..500 positives (score ≥ 4.0) → 30,491 users kept, 2,206,131 positives.
**Split:** per-user leave-last-10% of positives (standard LOO-style holdout). Global ordering by `created_at` on the full catalog turned out to partition *users* rather than time (only 1 user overlap between first-90% and last-10%), so the per-user protocol is used here and is not directly comparable to the 10K table above.
**Evaluation:** Full-catalog ranking against all 62,423 movies, train history masked per user.
**Positives evaluated:** 207,282 held-out val ratings from 30,491 users.

| Model                                    | HR@10  | HR@50  | HR@100 | NDCG@10 | NDCG@100 | MRR    |
|------------------------------------------|-------:|-------:|-------:|--------:|---------:|-------:|
| Random (theoretical, 62K cat.)           | 0.00016 | 0.00080 | 0.00160 | 0.00007 |   0.00023 | 0.00032 |
| Popularity                               | 0.0132 | 0.0579 | 0.0990 |  0.0058 |   0.0219 | 0.0075 |
| ALS (128 factors, 15 iters)              | 0.0238 | 0.0972 | **0.1645** |  0.0113 |   0.0377 | 0.0132 |
| Two-Tower v5 (history-BoW, genome, 256d) | 0.0251 | 0.0816 | 0.1283 |  0.0123 |   0.0319 | 0.0128 |
| **Two-Tower v5 + Cross-Encoder (top-200 re-rank)** | **0.0345** | **0.1045** | 0.1529 |  **0.0172** |   **0.0400** | **0.0168** |

v5 checkpoint: `lightning_logs/version_21/checkpoints/best-epoch=2-val_hr_at_10=0.3642.ckpt` (MLflow run `e33ec0f8cdf642b8b63816fee937ce83`, registered as `two_tower_recsys` v5).
Cross-encoder checkpoint: `lightning_logs/version_22/checkpoints/best-epoch=9-val_mse=0.7102.ckpt` (MLflow run `f292c829c1d740cf853ee58034b8e38f`, registered as `cross_encoder` v1). 4-MLP re-ranker on concat(u, i, u·i, |u−i|), trained pointwise on rating scores.

Key observations:
- **Multi-stage pipeline wins on 5 of 6 metrics.** HR@10 jumps 0.0251 → 0.0345 (+37 %), NDCG@10 0.0123 → 0.0172 (+40 %), MRR 0.0128 → 0.0168 (+31 %). The only remaining hole is HR@100 where ALS (0.1645) still beats the pipeline (0.1529) — expected, because re-ranking past position 200 does nothing, and deep recall favours MF's calibrated long-tail scores.
- **Two-Tower v5 alone overtakes ALS on top-K precision** (HR@10 / NDCG@10). Classical MF is still stronger at pulling long-tail positives anywhere in the top-100.
- **Popularity is beaten decisively by both models on every metric** at 5M / 62K scale (contrast with the 10K run where it was competitive with Two-Tower v2). Scaling the data gives personalised models their expected lift.
- **Re-ranking cost is bounded:** the cross-encoder sees only 200 pairs per query, so latency is constant regardless of catalog size.

Notes on protocol:
- The earlier 10K table used a global temporal split that, on this DB, actually partitions users (only 1 user overlap). `val_hr@10=0.36` during training was therefore measured on users the model had never updated. The per-user leave-last-10% split used here is the standard MovieLens LOO-style protocol and produces comparable numbers across all four models.

## What would improve the results

- **10x more users.** At 50K+ users, neural CF typically catches up to and surpasses MF baselines.
- **Hybrid scoring:** linearly combine Two-Tower and ALS scores at inference time.
- **Item2Vec / GraphSAGE** on the user-item bipartite graph as additional feature source.
- **Pre-train item embeddings** via GenSAGE before plugging into Two-Tower.
- **Knowledge distillation** from ALS into Two-Tower (use ALS scores as soft targets).

All of these are out of scope for this course project.
